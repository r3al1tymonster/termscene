the honest pitch

Why not just vibe-code your own?

You can fake a terminal div with a typing animation in twenty minutes. The hard part isn't the look — it's everything after.

vibe-coding it yourself
·Screen-record a CSS animation → dropped frames, jitter, a cursor caught mid-blink on the loop seam.
·The capture looks different every take — not reproducible.
·Change one word of narration → re-perform the whole take by hand.
·Eight on-brand, same-pacing clips? Eight separate fights.
·Fine for one GIF, once. The detour you were trying to avoid.
termscene
→Rendered as a pure function of time — byte-identical every run, any fps, scrubbable.
→Deterministic capture pipeline (headless Chrome + ffmpeg) is solved for you.
→Edit a line → re-render the identical-but-updated clip. No re-performing.
→Eight clips from eight scene files sharing one engine — consistent by construction.
→Earns its keep on the 2nd, 5th, 20th clip — and across a set.

The moat isn't the terminal look. It's deterministic video out of animated HTML — and an assistant that writes the scene for you.

{
  "meta": {
    "theme": { "preset": "claude" },
    "aspect": "wide",
    "window": { "chrome": "mac" }
  },
  "steps": [
    { "cmd": "add a /health endpoint" },
    { "out": "● Edit(server.ts)  +9", "style": "accent" },
    { "out": "{ status: ok }", "style": "ok" }
  ]
}

A scene is just steps on a clock.

cmd types a command. out prints output — with a style and optional char-by-char streaming. progress animates a bar. wait pauses. You never write timecodes; the compiler lays them out.

Then one command renders it — deterministically.
